yButton

Description​

The YButton is a clickable element with different optical styles, used to indicate its different states. The YButton can be used to trigger a user action. For example:

  • Calling a custom JavaScript function
  • Hiding and showing popups
  • Interacting with yeetClipboards

Basic Structure​

In the following the basic structure of the YButton shall be explained. For the general Structure of a YComponent please visit the YBase-components basic-structure.

The YButton as seen above contains a Label and a Icon.

Label​

The Label contains the text inside the button. It is normally in the center of the component, but can also be displayed on the right or left site. By changing the inner padding of the Button, the Label gets moved too. You can find further explanation at the Label Property.

Icon​

The Icon can be used to show a Icon inside the YButton. You can have a maximum of two Icons in a YButton. The Icon can be before or after the text and if you want to have only one YButton with one Icon, then you have to delete the text in the Label. You can find further explanation at the Icon Property. Properties​

Through its various properties the YButton can be configured to suit your needs. The Display below provides you with an overview of all the YButton properties.

Properties can be changed directly through three methods:

  • Inside the propertybar 1 and propertybar 2, which is positioned in your Workarea next to your component where you need it. It shows the most important properties, thus providing a fast and efficient way to configure your component in the most basic way.
  • It may also occur in the propertybar 2 which is a seamless extension accesible as a dropdown-item of the PB1. Extends the functionality of the PB1 by providing advanced Pickers for most used Properties.
  • Inside the propertybar 3, which is located in the righthand Drawer. Every Property of a component can be configured here.

Usage​

The following section provides detailed explanations of the various properties from the previous section and their usage. The headings and subheadings serve as the categorization into Categories and Groups. Only the properties that are relevant to are listed below. General properties that are applicable universally can be found in the YBase-component's usage-section.

Style​

Style > Label​

The Label can be used to show text inside the YButton. You can only have one Label in a YButton. If you want to customize it, you can do this with the TypographyPicker in propertybar 1 and propertybar 2 or under the Style Menu in the propertybar 3.

font-color​

ColorProperty

The color of the Label. This can be set as a color from a palette or a custom hex color. You can choose a color via the ColorPicker.

font-family​

ListProperty

The font-family of the Label. This can be set to a font-family from the specified list of fonts. The preselected font-family is "Quicksand".

font-weight​

NumberProperty

With the font-weight of the Label you can choose how bold your font should be. Normally font-weight is set in 100 steps. 100 is very thin, 900 very thick. The normal font-weight is 400.

font-size​

UnitNumberProperty

The font-size of the Label. This can be set in:

cm mm in px em ex ch rem

font-style​

ListProperty

The font-style of the Label. This can be set in:

normal italic oblique

text-transform​

ListProperty

You can transform the text of the Label with the text-transform property. Here you can set whether the text should automatically be written in upper or lower case and much more. The options are:

capitalize uppercase lowercase full-width none

font-align​

ListProperty

The font-align of the Label. Here you can determine whether the text should be aligned left-right or centered. This can be set in:

left center right

label​

StringProperty

With this property you can decide what should be the actual text that will be displayed in the label.

tip

The label text can also be set through the inplace-edit. Simply double click the components and the edit will appear.

Style > Icon​

To change the icon of a button you need to change the icon inside the icon picker of propertybar 1 and propertybar 2 or under the Style Menu in the propertybar 3.

icon-position​

ListProperty

The button has built-in support for 4 icon-positions: left, right, both, none The icon color can be set through the icon-primary-Color property.

With this property you can choose on which side of the button the icon should be displayed. If you want your button to have two icons, select iconBoth here. The default is none, which means that there is no icon in the button at all.

none iconLeft iconRight iconBoth

tip

If you want to have a button without text, only with an icon, you have to leave the label property empty under Style > Label > label.

icon-color-primary​

ColorProperty

The primary color of the Icon. This can be set as a color from a palette or a custom hex color. You can choose a color via the ColorPicker.

icon-color-secondary​

ColorProperty

The secondary color of the Icon. You need to set this property if you have a symbol with two possible color slots and you want each color to be different. It can be set as a color from a palette or a custom hex color. You can choose a color via the ColorPicker.

icon-size​

UnitNumberProperty

The icon-size of the Icon. This can be set in:

cm mm in px

icon-left​

IconProperty

With this property you choose which Icon you want to show at the icon left position. With the Icon Picker you can select your desired icon.

The propertybar 2 IconPicker works as follows:

  • First you have to select whether you want to edit a right or a left icon. This is done by activating the R- or L-Button. If you want to have two icons and edit them at the same time, you can also activate R and L at the same time.
  • After you have selected the desired icon position, you can select an icon library using the following selectionpicker. (The icons from Font Awesome are preselected.)
  • After that you can select one from the list of icons shown below by clicking on it. Furthermore you have the possibility to search for specific icons.
  • As a last step you can set the icon color on the right side. This picker works very similar to the color picker.

To configure the Icon via the propertybar 3 Icon-group works almost the same way, except that you have to open a separate picker for the icon_left and icon_right by clicking on the "Select"-Button. Also you have to set the icon color individually under the properties icon_color_primary (for icon_left) and icon_color_secondary (for icon_right).

Button​

In this section everything related to the general behavior of the YButton can be set.

Button > State​

With the properties of the State-group you can decide if the YButton is in an active or disabled state. These properties are active and disabled.

Button > Tooltip​

When you write something in the tooltip property of the Tooltip-group the YButton will show your input as a Tooltip, which is shown when you hover over the YButton.

Button > Misc​

In the Misc-group are properties, which influence the behavior of the YButton. If you activate toggle-mode property, your button will behave like a toggleButton, which you can toggle on and off.

Events​

In this section everything related to events can be set. The custom function calls and remote procedure calls are also to be made in this section.

Events > Control​

This is where you can manage the events associated with the clipboards. However, it's crucial to ensure that these events are activated. Without this, the button won't be able to trigger the desired action on the clipboard.

Events > Input​

This section allows you to access the editor for defining the YButton's click function. Within this editor, you can write custom JavaScript code, which gets executed whenever the YButton is clicked. For practical application of this feature, refer to the use case example provided below.

Examples of use​

Calling a custom JavaScript function​

The Button component enables you to execute a custom JavaScript function each time it's clicked.

For instance, you could set up a function that creates a counter on the button, incrementing each time the button is pressed. Once the counter reaches the number 10, it automatically resets. The steps to achieve this are detailed below.

How to Use​

  1. Create a YButton component in your application.
  2. Select the created YButton and open the propertybar 3.
  3. Open the editor under events -> input -> Click-Function
  4. Associate a custom JavaScript function to the YButton. This function will be invoked every time the button is clicked.
  5. Include the provided JavaScript snippet below to implement a counter function on the button.

Code Example​

You can add the following JavaScript code snippet to your application to implement a counter function on the YButton component.

With this script, each click on the YButton will increment the displayed counter until it reaches 10, upon which it will display a congratulatory message and reset. The button's background color will also change depending on the counter's current state.

Custom Button-Click Function
// Properties names are shown in tool tips of designer property bar.
// Get "label" property of button object (category = style, group = label, name = label).
let buttonText = obj.get("style", "label", "label");
// Convert string to number.
let counter = Number(buttonText);
// Not a number -> reset counter.
if (Number.isNaN(counter)) {
  counter = 0;
  obj.set(
    "style",
    "background",
    "bgColor__normal",
    new api.types.colorValue("#22aaffFF")
  );
  obj.set(
    "style",
    "background",
    "bgColor__hover",
    new api.types.colorValue("#49B9ffFF")
  );
}
// Increment counter.
else {
  counter++;
}
// Count until ten.
if (counter == 10) {
  buttonText = "Well done!";
  obj.set(
    "style",
    "background",
    "bgColor__normal",
    new api.types.colorValue("#00ff0080")
  );
  obj.set(
    "style",
    "background",
    "bgColor__hover",
    new api.types.colorValue("#00ff00FF")
  );
} else {
  buttonText = counter.toString();
}
// Log counter value in browser console.
console.log(counter);
// Set property value of button.
obj.set("style", "label", "label", buttonText);

Hiding and showing Popups​

You can use the YButton to open and close popups.

Here's how you can implement this:

  1. Create a YButton component in your application.

  2. Navigate to the popup you wish to control and open its propertybar 3 settings.

  3. Within these settings, proceed to the "Links" section and select "Open".

  4. In this area, use the SourceLink Picker to select the button you created earlier. As event choose "click".

And thats it, you can now test this functionality by by entering the preview mode and clicking on the button to observe the popup's visibility change. Visit the Popup section, if you want to know more about their behavior.

Interacting with yeetClipboards​

You can use the YButton to control Clipboards. The YeetClipboard functionality provides an effective method for managing for example table entries. As example, you can set up a mechanism to delete a table entry by a simple button click. Here's how to accomplish this:

  1. Create a YButton component in your application.
  2. Select the created YButton and open the propertybar 3.
  3. Proceed to the Events tab and locate the "Control" section. Here, you'll need to enable the "Commit-Create" option. This step empowers the YButton component with the capability to perform this functionality.

  1. Go to the YeetClipboard[LINK!!]and create a new Clipboard. And enable the "Use Entity" toggle.
  2. Than you can click on "Entity-Properties" and edit the "Entity". This clipboard should be linked to the entity containing the table entry you wish to delete. So you have to fill in the data, like in the picture below.

  1. The _id under "Keys" should be the entry that you wish to delete.
  2. Finally, go to the "Events", edit it and select the Button as SourceLink-1 via the SourceLink-Picker. Dont forget that the Event has to be "commit-delete".

This will establish a button that, when clicked, will delete the specified entry from your table.

If you want to know more about this topic, visit the YeetClipboard documentation page [LINK!!].
