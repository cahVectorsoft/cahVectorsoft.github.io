Propertybar 1

The Propertybar 1 is the most important propertybar. It contains almost all the functionality to adjust a component of your choice to your needs. The Propertybar 1 has two kinds of entries: toggle entries, which activate an action or toggle a state, and dropdown entries, which show and hide a propertybar 2 picker.

Behavior​

Placement​

The propertybar 1 if not pinned will appear dynamically at the best possible place inside the workarea. The propertybar 1 will at first try to place itself above the selected component, while still providing enough space for child-menus to open without leaving the window and also not leaving the workarea. If that's not possible the propertybar 1 will try to place itself below the component. If even that is not possible it will be displayed inside the component at the top.

Drag n' Drop​

The propertybar 1 can be moved around and placed where needed by clicking the left mouse-button it somewhere at the border, moving it somewhere and releasing the right mouse-button.

Pin​

The propertybar 1 can be pinned by clicking the pin-needle-icon in the top right corner. The icon will appear when hovering the top-right corner. When activated the icon will stay visible and the propertybar 1 will stay in place even when changing the active component or trying to reposition it. For the position to be changed again and unpin the propertybar 1 the icon has to be clicked again.

PB3-Button​

The pb3-button is present in every propertybar 1. The button will toggle the propertybar 3 on or off.

Normal Propbertybar 1​

The normal Propertybar One is the Propertybar that opens up under, over or inside a newly selected component. The normal Propertybar One always contains "spacing", "sizing", "state", "background-color", "shadow", "border", "delete" and "open Propertybar 3".

Sizing​

Enable the Inplace-Sizing Mode through clicking the sizing icon in the Propertybar1. Afterwards you can adjust the size through Dragging the corresponding handler.

Spacing​

Enable the Inplace-Spacing Mode through clicking the spacing icon in the Propertybar1. Afterwards you can adjust either the margin or padding values, default is Margin, through Dragging the corresponding handler.

Additionally you can activate the lock-mode, with this option enabled adjusting one Margin- or Padding-Side, will also change all other sides.

Switch between Margin and Padding through the Propertybar1.

Enable the Colour Picker through clicking the color icon in the Propertybar1. In the resulting PB2 you can configure the background color aswell, as its opacity.

Theme: In case you are working with YeetThemes simply choose the correct theme Level, your Theme-style-definition will be applied to the component-border. ThemeColors: In case you don't want to create a specific theme definition for your component but still want to use a predefined color-set

  • BasicColors: A sample of basic colors which you can use
tip

The Colour of the Background icon equals the Background-Colour setting

Border​

Enable the Border Picker through clicking the border icon in the Propertybar1. In the resulting PB2 you can configure the Border radius, size, color and style

The first option which has to be enabled in order to see a border is the border-style property, located at the top of the Picker. Currently there are 4 styles supported: None (Default), Solid, Dashed and Dotted.

You can adjust the Border thickness through the slider positioned directly beneath the style options, you will have to choose a thicknes > 0 in order to see a border.

The Opacity slider enables you to adjust the transparency of the border, where 0 = Completely visible and 100 = Completely transparent.

Situated beneath the opacity slider is a Color Picker which sahres many similarities with the BackgroundColor Picker

tip

The Colour of the Border icon equals the Border-Colour setting

Shadow​

Enable the Shadow Picker through clicking the shadow icon in the Propertybar1. In the resulting PB2 you can configure the components shadow, like thickness, color, style and opacity.

x and y​

The x and y values will move the whole shadow. For positive x-values the shadow will be moved to the right, for negative ones to the left. For positive y-values the shadow will be moved to the bottom, for negative ones to the top. The unit of the values is in all cases pixels.

blur​

The blur value is also given in pixels and will blur the shadow depending on the value, the higher the value the more intense the blur. The value has to be positive, negative values are forbidden.

spread​

The spread value is also given in pixels. This value is responsible for making the shadow, which by default is the same width and height as its component, bigger or smaller. A positive value will increase the width and height of the shadow a negative value will decrease it.

tip

The Colour of the Shadow icon equals the Shadow-Colour setting

State​

Switch the current state of the component, this will also apply to the Propertybar3, also works in reverse (switching the Sate in the PB3 will also switch the Component and the Pb1). The component will be displayed in the specified state.

Most Style properties exist in each state configuration provided by the component.

The following states are present in every component: Normal, Hover, Active, Focus, Disabled

Complete list of States. Note: Not every component has every state. Lookup the component you are interested in to see which states are available.

StateNormal activation
NormalDefault State, no interaction
HoverPoint the Mouse at the Component no Click
FocusComponent is clicked / Selected through Tab
ActiveComponent was clicked (primary Mouseboutton only)
DisabledComponent is Disabled (Programatic Feature)
Error(Propgramatic Feature) Example: Set when input should have content but is empty

tip

Programatic Feature don't have a natural origin. Instead they'll only be applied when specifically set.

Special Propertybar 1​

Certain Pb1-pickers will open an extensions (PB2) which provides Smarter Pickers, as well as extended configuration options. The special Propertybar One is a hidden mode of the Propertybar One that can be activated for extended functionality of a component. For Example in the YTextbox, YInput, YLabel and others.

In order to return to the default Propertybar1 either click the Backicon (left Arrow, lefthand side) or simply de- and reselect the component.

Propertybar 2

The Propertybar 2 can be reached by selecting a dropdown entry in the Propertybar 1.

TypographyPicker​

The Typographypicker is present in multiple components. The TypographyPicker is responsible for various styling options for text-related content.

At the top there is a font-family-picker as a LINK-SELECTIONPICKER, followed by an SML and left-right-center-picker both as LINK-TOGGLE-PICKERS. Then there is a subthemepicker as a LINK-SELECTIONPICKER
